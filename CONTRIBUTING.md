# Contributing to Red Dawn

Thank you for your interest in contributing to Red Dawn! We welcome all contributions.

## How to Contribute

### 🐛 Bug Reports
1. Check if issue already exists
2. Include detailed description
3. Steps to reproduce
4. Your environment (OS, Stremio version)
5. Screenshots if applicable

### ✨ Feature Requests
1. Describe the feature
2. Explain the use case
3. Any implementation ideas
4. Examples of similar features

### 📝 Code Contributions

#### Setup Development Environment
```bash
git clone https://github.com/Tre9995/reddawn-addon
cd reddawn-addon
npm install
npm start
```

#### Making Changes
1. Create feature branch: `git checkout -b feature/your-feature`
2. Make your changes
3. Test thoroughly
4. Commit with clear messages: `git commit -m "Add feature description"`
5. Push to branch: `git push origin feature/your-feature`
6. Open Pull Request

#### Code Style
- Use 2-space indentation
- Use camelCase for variables
- Add comments for complex logic
- Keep functions small and focused
- Test your code

#### Testing
```bash
npm test
npm run lint
```

### 📚 Documentation
- Fix typos
- Improve clarity
- Add examples
- Update outdated info
- Translate to other languages

## Pull Request Process

1. Update README.md with changes
2. Update version in manifest.json
3. Provide detailed PR description
4. Link related issues
5. Ensure all tests pass
6. Wait for review

## Code of Conduct

- Be respectful and constructive
- No harassment or discrimination
- Focus on the code, not the person
- Help others learn
- Report violations to maintainers

## Questions?

- Check existing issues/discussions
- Ask in GitHub Discussions
- Contact: support@reddawn.xyz
